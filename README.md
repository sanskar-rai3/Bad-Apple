## Dependencies

- **OpenCV** — video decode and frame processing
- **SDL2** — audio backend
- **SDL2_mixer** — MP3 playback

On Arch Linux:
```bash
sudo pacman -S opencv sdl2 sdl2_mixer
```

On Ubuntu/Debian:
```bash
sudo apt install libopencv-dev libsdl2-dev libsdl2-mixer-dev
```

## Assets

The video and audio files are not included due to copyright. Place the following files in the `asset/` directory yourself:

```
asset/
├── bad_apple.mp4
└── bad_apple.mp3
```

The program tries both `asset/` and `../asset/` so it works when run from either the project root or a `build/` subdirectory.

## Build

```bash
mkdir build && cd build
cmake ..
make -j$(nproc)
```

## Run

From the `build/` directory:
```bash
./Bad-Apple
```

Or from the project root:
```bash
./build/Bad-Apple
```

## Terminal Setup

For best results:

- Use a terminal that supports ANSI escape codes (virtually all modern terminals do)
- A dark background with a light foreground gives the most contrast

## How It Works

Video and audio run on separate threads. The audio thread initializes SDL2, loads the MP3, and starts playback — recording the exact wall-clock timestamp at that moment. The video thread blocks until audio is confirmed playing, then computes which frame it should be displaying based on `elapsed_time × fps` rather than counting sleeps. This means:

- No startup desync from SDL2 init overhead
- Drift is corrected every frame rather than accumulating
- If rendering falls behind, frames are skipped to catch up

Each frame is converted to grayscale, scaled to 160×45, and each pixel is mapped to a character from ` .,-~+=&%#$@` by brightness. Characters are doubled horizontally to compensate for the ~2:1 height-to-width ratio of monospace fonts.
